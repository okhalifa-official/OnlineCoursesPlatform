## 7. Other Non-Functional Attributes

| Identifier | Attribute             | Requirement                                                                                                                                                                                                                                                                                                                                                                     |
| ---------- | --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| NFR-001    | Security              | The system shall protect restricted pages from unauthenticated users. Applied in the project, Login authentication is used before users can access protected dashboards.                                                                                                                                                                                                        |
| NFR-002    | Reliability           | The system shall continue operating when an API request fails. Apllied in the ptoject, Failed requests show error messages instead of breaking the whole interface.                                                                                                                                                                                                             |
| NFR-003    | Maintainability       | The system shall separate frontend code from backend code. Apllied in the project, The project has separate frontend and backend folders.                                                                                                                                                                                                                                       |
| NFR-004    | Portability           | The system shall run on modern web browsers. Apllied in the project, The system runs as a web-based MERN application.                                                                                                                                                                                                                                                           |
| NFR-005    | Extensibility         | The system shall allow new pages to be added without rebuilding the whole project. Apllied in the project, New dashboard pages can be added through React routing and separated components                                                                                                                                                                                      |
| NFR-006    | Re-usability          | The system shall reuse shared interface components. Apllied in the project, Navigation bars, forms, buttons, cards, and API helper files are reused.                                                                                                                                                                                                                            |
| NFR-007    | Application affinity  | The system shall follow the selected MERN-stack architecture. Applied in the project, The project uses MongoDB for the database, Express.js for backend routing, React.js for the frontend interface, and Node.js for the backend runtime.                                                                                                                                      |
| NFR-008    | Resource utilization  | The system shall request only needed data for each page. Applied in the project, Pages load data based on their function instead of loading all system data at once.                                                                                                                                                                                                            |
| NFR-009    | Serviceability        | The system shall allow developers and Admin users to identify, trace, and diagnose system issues. Apllied in the project, The system includes system logs for important actions such as create, update, archive, restore, and delete. Backend errors can also be traced through logged route and request information, but advanced service tools are not fully implemented yet. |
| NFR-010    | Response Time         | The system shall load dashboard data within 1000 ms under normal local testing conditions. Apllied in the project, Dashboard data is loaded through API requests. Formal response-time testing is not completed yet.                                                                                                                                                            |
| NFR-011    | Authentication        | The system shall require each user to log in before accessing private pages. Apllied in the project, Admin, Instructor, and Student users must log in before entering their dashboards.                                                                                                                                                                                         |
| NFR-012    | Authorization         | The system shall restrict each user role to its assigned permissions. Apllied in the project, Role-based access is used for Admin, Instructor, and Student users.                                                                                                                                                                                                               |
| NFR-013    | Password Protection   | The system shall store user passwords in protected form. Apllied in the project, Password protection is handled in the backend authentication logic.                                                                                                                                                                                                                            |
| NFR-014    | Session Management    | The system shall maintain user sessions using authentication tokens. Apllied in the project, Tokens are used for logged-in users. Token expiry needs final confirmation.                                                                                                                                                                                                        |
| NFR-015    | Availability          | The system shall remain accessible while the frontend server is running. Apllied in the project, The system runs locally through the frontend server. Production uptime is not tested yet.                                                                                                                                                                                      |
| NFR-016    | Error Handling        | The system shall display an error message when a request fails. Apllied in the project,Error messages appear during failed login, failed form submission, or failed data loading.                                                                                                                                                                                               |
| NFR-017    | Input Validation      | The system shall validate required fields before saving records. Apllied in the project, Required fields are checked before saving users, courses, and forms.                                                                                                                                                                                                                   |
| NFR-018    | Data Integrity        | The system shall prevent incomplete required records from being stored. Apllied in the project, MongoDB models and backend validation help protect stored data.                                                                                                                                                                                                                 |
| NFR-019    | Performance           | The system shall load page data through API requests. Apllied in the project, Pages request data from backend endpoints. Advanced performance testing is not completed yet.                                                                                                                                                                                                     |
| NFR-020    | Scalability           | The system shall allow new users to be added without changing the system structure. Apllied in the project, The MERN structure supports adding more users. Large-scale load testing is not completed yet.                                                                                                                                                                       |
| NFR-021    | Modularity            | The system shall divide features into independent modules. Apllied in the project, Users, courses, payments, reports, notifications, and logs are organized as separate modules.                                                                                                                                                                                                |
| NFR-022    | Accessibility         | The system shall use readable labels on main buttons. Apllied in the project, Buttons and forms use visible labels. Full accessibility testing is not completed yet.                                                                                                                                                                                                            |
| NFR-023    | Audit Trail           | The system shall allow Admin users to review system changes. Apllied in the project, Admin users can view logged actions through the system logs page.                                                                                                                                                                                                                          |
| NFR-024    | Audit Trail           | The system shall allow Admin users to review system changes. Apllied in the project, Admin users can view logged actions through the system logs page.                                                                                                                                                                                                                          |
| NFR-025    | Logging               | The system shall record important system actions. Apllied in the project, System logs track create, update, archive, restore, and delete actions.                                                                                                                                                                                                                               |
| NFR-026    | Backup                | The system shall support automated database backup. Apllied in the project, MongoDB stores the data, but automated backup is not implemented yet.                                                                                                                                                                                                                               |
| NFR-027    | Configuration         | The system shall store sensitive configuration outside frontend source code. Apllied in the project, API URLs, database connection strings, and secret keys are handled through configuration values.                                                                                                                                                                           |
| NFR-028    | Deployment Readiness  | The system shall separate frontend deployment from backend deployment. Apllied in the project, The frontend and backend are separated. Final production deployment is not completed yet.                                                                                                                                                                                        |
| NFR-029    | Interoperability      | The system shall exchange data using JSON format. Apllied in the project, The frontend communicates with the backend through REST APIs using JSON.                                                                                                                                                                                                                              |
| NFR-029    | API Consistency       | The system shall use structured API routes for system features. Apllied in the project, Routes exist for users, courses, reports, payments, settings, notifications, and logs.                                                                                                                                                                                                  |
| NFR-030    | Privacy               | The system shall protect user account data from unauthorized access. Apllied in the project, Authentication and role-based access protect user accounts.                                                                                                                                                                                                                        |
| NFR-031    | Data Storage          | The system shall store platform records in MongoDB. Apllied in the project, Users, courses, lectures, payments, reports, notifications, and logs are stored in MongoDB.                                                                                                                                                                                                         |
| NFR-032    | Data Retrieval        | The system shall retrieve stored records through backend endpoints. Apllied in the project, Frontend pages retrieve MongoDB data through Express API routes.                                                                                                                                                                                                                    |
| NFR-033    | Frontend Consistency  | The system shall use consistent visual styling across main pages. Apllied in the project, Pages use consistent layouts, buttons, forms, and dashboard cards.                                                                                                                                                                                                                    |
| NFR-033    | Backend Consistency   | The system shall organize backend logic using consistent folders. Apllied in the project, Backend code is organized into routes, controllers, models, and middleware.                                                                                                                                                                                                           |
| NFR-034    | Role Separation       | The system shall separate Admin access from User access. Apllied in the project,Admin pages are protected from non-admin users.                                                                                                                                                                                                                                                 |
| NFR-035    | Admin Control         | The system shall allow Admin users to manage platform records.Apllied in the project,Admin users can manage users, courses, reports, payments, notifications, and logs.                                                                                                                                                                                                         |
| NFR-036    | Payment Configuration | The system shall allow payment settings to be managed. Apllied in the project, Payment settings exist. Real payment gateway integration is not fully completed yet.                                                                                                                                                                                                             |
| NFR-037    | Notification Support  | The system shall support notification-related features. Apllied in the project, Notification features exist. Full email notification delivery is finalized.                                                                                                                                                                                                                     |
| NFR-038    | Searchability         | The system shall allow records to be displayed in organized lists. Apllied in the project, Users, courses, and logs are displayed in organized pages. Advanced search may need more testing.                                                                                                                                                                                    |
| NFR-038    | Data Consistency      | The system shall keep related data organized through database models. Apllied in the project, MongoDB models help keep related data structured.                                                                                                                                                                                                                                 |
| NFR-039    | System Traceability   | The system shall allow system changes to be traced. Apllied in the project, System logs allow Admin users to trace important changes.                                                                                                                                                                                                                                           |
